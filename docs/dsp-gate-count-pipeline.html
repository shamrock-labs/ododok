    변수 사전

    변경 전 이름과 현재 코드 이름을 나란히 두어 책임을 명확히 합니다.

    Queue · 입력 순서

    Queue는 Gate를 판단하지 않습니다. 센서 행이 DSP에 순서대로 들어가도록 보장합니다.

    변경 전chewCounterFeedTask
    현재sampleProcessingTailTask
    입력 함수enqueueChewDetectionSample
    종료 규칙마지막 Task가 끝난 뒤 finishSession()

    Gate · 활동 구간

    실제 씹기 횟수가 아니라, 지금 peak를 세도 되는 구간인지를 나타냅니다.

    isChewingisChewingGateOpen
    chewingLikematchesChewingGate
    enterSampleCountsamplesRequiredToOpen = 10
    exitSampleCountsamplesRequiredToClose = 30

    Peak · 후보와 대표값

    Gate가 열려 있을 때만 local peak를 받고, 0.3초 창 안에서 가장 강한 값을 선택합니다.

    isPeakisLocalPeakCandidate
    RepresentativePeakSelectorRepresentativePeakWindow
    representativePeakSelectorpeakSelectionWindow
    최소 진폭minPeakAmplitude = 0.006
    선택 창windowDuration = 0.30 s · 약 15행

    Count · 확정과 UI

    확정 Count는 데이터이고 UI Pulse는 피드백입니다. UI가 합쳐져도 Count는 보존합니다.

    chewCountconfirmedChewCount
    registerconfirmChew
    최소 간격minPeakGapSeconds = 0.32 s
    UI 합치기minimumInterval = 0.20 s
    모델 버전dsp-chewcounter-2
    이 시뮬레이터는 Gate의 Biquad·EWMA 계산을 전부 재현하지 않고, 그 계산 결과인 matchesChewingGate를 입력값으로 사용합니다. 실제 코드에서는 rotation 표준편차, Y축 우세도, jitter 우세도, 가속도 대비 회전 에너지가 먼저 계산됩니다.
